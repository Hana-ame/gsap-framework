# Pixi.js React 测试项目 (带时间戳事件日志)

## 一、项目概述与上下文

本项目是一个基于 **PixiJS v8** 和 **React** 的测试框架，采用**解耦的控制器-插件架构**。主要目标是通过 React 组件方便地控制 Pixi 画布进行绘图，并实时接收画布上的用户交互事件（带时间戳），以便调试和分析。

### 核心架构

- **`PixiController`**：消息中枢。负责管理插件、分发绘图指令，并将画布事件（如点击、移动）传递给 React 父组件。所有消息均包含时间戳。
- **`PixiCanvas`**：React 组件。封装 Pixi 应用的生命周期（初始化、销毁），监听画布上的鼠标/触摸事件，并通过 Controller 发送带时间戳的事件。
- **插件系统**：独立的绘图插件（如画圆、画矩形），每个插件声明自己能处理的消息类型。未来可扩展 WebSocket 数据源。

## 二、当前版本功能 (Version 2.0)

### ✨ 新增功能

1.  **带时间戳的事件日志**：所有从画布发出的事件（`click`, `pointerdown`, `pointermove`, `pointerup`, `pointerleave`）都包含 `timestamp` 字段，可在 UI 上实时显示，便于追踪操作时序。
2.  **增强的交互监听**：
    - **按下 (pointerdown)**：鼠标/触摸按下时触发。
    - **移动 (pointermove)**：在画布内移动时持续触发。
    - **弹起 (pointerup)**：释放鼠标/触摸时触发。
    - **离开 (pointerleave)**：指针移出画布区域时触发。
    - **点击 (click)**：保留作为额外事件。
    - 支持鼠标和触摸事件，坐标已转换为画布相对坐标。
3.  **测试界面升级**：新增“事件日志”区域，按时间倒序显示最近 50 条事件，并附带坐标信息。同时增加“清除日志”按钮。

### ✅ 继承保留的功能

- 插件注册与消息分发机制。
- 通过按钮发送绘图指令（画圆、矩形、清除）。
- 画布生命周期管理（自动清理，防止内存泄漏）。

## 三、使用方法

1. **安装依赖**:

    ```bash
    npm install pixi.js@8.x react react-dom
    ```

2. **运行项目**:

    ```bash
    npm run dev
    ```

3. **测试交互**:
    - **点击画布**：在画布上点击，日志区会显示 `click` 事件和时间、坐标。
    - **在画布上按下并移动**：依次触发 `pointerdown` -> 多个 `pointermove` -> `pointerup`（或移出画布触发 `pointerleave`）。
    - **触摸设备**：支持触摸事件，操作类似。
    - **点击按钮**：发送绘图指令，同时日志区记录发送指令的动作。

## 五、需要删除或弃用的测试项

根据新功能，以下旧的测试项可以移除或不再关注：

1. **旧版“点击canvas success?”测试**：已被更详细的事件日志覆盖，无需单独验证。
2. **旧版“点击按钮 success?”测试**：同样被绘图指令日志和视觉反馈覆盖。
3. **无时间戳的日志输出**：所有日志现在强制包含时间戳，旧的不带时间戳的打印方式（如有）应移除。

## 六、注意事项

- **时间戳精度**：使用 `Date.now()` 获取毫秒级时间戳，在日志中格式化为 `HH:MM:SS.mmm`。
- **性能**：`pointermove` 事件可能高频触发，日志更新已限制最多保留 50 条，避免界面卡顿。
- **坐标限制**：事件坐标被限制在画布尺寸内（0 ≤ x ≤ width, 0 ≤ y ≤ height），防止负值或超出。
- **触摸事件**：已添加 `preventDefault()` 防止触摸时页面滚动，可根据需要调整。
- **插件执行**：当前示例中绘图指令未完全通过控制器统一调度，而是直接调用插件。正式使用时建议完善控制器的 `sendToPixi` 方法，使其能够接收 `app` 实例或通过依赖注入。

## 七、未来可扩展方向

- 完善控制器的插件执行机制，统一管理 `app` 实例。
- 接入 WebSocket，将外部消息转换为绘图指令或事件。
- 增加事件节流/防抖，优化高频移动事件的日志显示。
- 支持拖拽图形等更复杂的交互。


# Pixi.js React 测试项目 (带时间戳事件日志)

## 一、项目概述与上下文

本项目是一个基于 **PixiJS v8** 和 **React** 的测试框架，采用**解耦的控制器-插件架构**。主要目标是通过 React 组件方便地控制 Pixi 画布进行绘图，并实时接收画布上的用户交互事件（带时间戳），以便调试和分析。

### 核心架构
- **`PixiController`**：消息中枢。负责管理插件、分发绘图指令，并将画布事件（如点击、移动）传递给 React 父组件。所有消息均包含时间戳。
- **`PixiCanvas`**：React 组件。封装 Pixi 应用的生命周期（初始化、销毁），监听画布上的鼠标/触摸事件，并通过 Controller 发送带时间戳的事件。
- **插件系统**：独立的绘图插件（对象字面量形式），每个插件声明自己能处理的消息类型，并包含具体的绘图逻辑。插件是函数式的，无需实例化。

## 二、技术指南：逻辑流程

### 1. 初始化阶段
- `App.tsx` 创建 `PixiController` 实例。
- 注册插件：调用 `controller.registerPlugin(pluginObject)`，传入符合 `PixiPlugin` 接口的对象（如 `circlePlugin`）。
- 设置父组件消息处理器：`controller.onMessageFromParent(handler)`，该处理器负责将画布事件显示在 UI 日志中。
- 渲染 `PixiCanvas` 组件，传入 `controller` 和 `onAppInit` 回调。

### 2. 画布初始化
- `PixiCanvas` 内部创建 `PIXI.Application` 并异步初始化。
- 初始化完成后，将 `app` 实例通过 `onAppInit` 回调传回 `App.tsx`。
- `App.tsx` 调用 `controller.setApp(app)`，使控制器持有 `app` 引用，以便后续插件执行绘图。

### 3. 用户交互事件流
- 用户在画布上操作（鼠标/触摸），`PixiCanvas` 监听相应事件（`pointerdown`、`pointermove` 等）。
- 事件处理器构造消息对象：包含 `type`、坐标 `x`/`y`、`timestamp`（`Date.now()`）等。
- 调用 `controller.sendToParent(message)` 将消息发送给父组件。
- 父组件的处理器格式化日志并更新 UI。

### 4. 绘图指令流
- 用户点击 UI 按钮（如“画圆”），`App.tsx` 调用 `sendDrawCommand('drawCircle')`。
- 构造绘图消息对象，包含 `type`、随机坐标、颜色等。
- 调用 `controller.sendToPixi(message)`。
- 控制器遍历已注册插件，寻找 `messageTypes` 包含该 `type` 的插件。
- 找到后调用插件的 `execute(message, app)` 方法，在画布上绘制图形。
- 若没有插件处理该类型，控制台输出警告。

### 5. 清理阶段
- `PixiCanvas` 组件卸载时，自动销毁 `PIXI.Application` 实例，释放资源。
